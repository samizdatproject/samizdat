SAMIZDAT BSV anonymous publishing protocol — community directory
About this directory

SAMIZDAT is a protocol for anonymous, non-custodial, onion-first content publishing anchored on the BSV blockchain. This page lists known renderers and indexers run by the community.

This directory is not authoritative. No entry here is more “official” than any other. Content is verified cryptographically by the renderer — not by this list.

Renderers

A renderer fetches a publication by txid, verifies all chunk hashes against the Merkle root, and serves the content. Any renderer that passes all hash checks is trustworthy by the protocol design — you do not need to trust the operator.

Indexers

An indexer scans the BSV chain for SAMIZDAT anchor transactions and exposes a search API. Indexers are explicitly non-canonical — their results are convenience pointers, not proof of publication. Proof is always the on-chain anchor.

! Privacy warnings for new publishers
Read before publishing.
  • Publication is permanent and irreversible. Once anchored on-chain, the content hash and all chunk data are publicly accessible forever.
  • Use Tor Browser. Open the editor from an onion address. Your IP address should never be associated with your content.
  • Use an onion-accessible BSV wallet. Transaction broadcasts expose your network location. Sign and broadcast from within the Tor network.
  • Strip metadata before uploading. The editor strips JPEG/PNG EXIF and PDF /Info fields automatically. For other formats (Office documents, audio/video), strip metadata manually with ExifTool before uploading.
  • Transaction timing and size are observable. A chain observer can correlate transactions to your content. Avoid publishing at predictable times or from a wallet linked to your identity.
  • Filenames are public. All filenames in the manifest are stored in plaintext on-chain. Do not use identifying filenames.
  • The title field is public. The manifest title (if set) is stored in plaintext on-chain. Leave it blank or use a pseudonymous title.

For a full threat model and operational security guide, see docs/opsec-guide.md.

How to add your endpoint

This directory is a community-maintained static HTML file. To list your renderer or indexer:

  1. Fork the SAMIZDAT repository
  2. Edit directory/index.html — add an <div class="endpoint-card"> in the appropriate section
  3. Include: name, address, badge type (onion / clearnet / self-hosted), brief description of your privacy policy
  4. Open a pull request
Listing requirements: no logs of txid queries or reader IPs; no user accounts; no tracking. Operators who cannot meet these requirements should not be listed.
Running your own node

You do not need a listing here to use SAMIZDAT. Any author can publish using the editor, and any reader can retrieve content from any renderer by knowing the txid. The protocol has no canonical gateway.

See docs/deployment.md for a complete guide to running your own renderer, indexer, or editor on an onion service.
